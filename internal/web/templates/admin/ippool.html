How concurrent users are handled

If 10 users each submit 5 emails (50 total):

  1. All 50 messages land in the queue instantly
  2. Workers pick them up simultaneously in round-robin order
  3. IP rotation distributes load across pool IPs
  4. Each IP respects its own per-min/hour/day limits
  5. If all IPs hit limits, falls back to system default IP
