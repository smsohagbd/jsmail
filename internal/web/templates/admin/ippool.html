How concurrent users are handled

If 10 users each submit 5 emails (50 total):

  1. All 50 messages land in the queue instantly
  2. Up to 150 workers pick them up simultaneously
  3. All 50 can deliver in parallel — no waiting
  4. IP rotation distributes load across pool IPs
  5. Each IP respects its own per-min/hour/day limits
  6. If all IPs hit limits, falls back to system IP
