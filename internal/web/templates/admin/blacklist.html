How it works: IP blacklists are queried via reverse-DNS lookup (e.g. 4.3.2.1.zen.spamhaus.org). Domain blacklists are queried as yourdomain.dbl.spamhaus.org. An A-record response means listed; NXDOMAIN means clean. All checks run in parallel using free DNS — no API key required.

What to do if listed: Visit the DNSBL's website to check the reason and submit a delisting request. Common causes: open relay, spam history, residential IP range (use a business datacenter IP), or shared hosting abuse.
